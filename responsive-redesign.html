Responsive Redesign

The Objective:

The Asmara Resturant is an establishment serving Eritrean cuisine, and owned by a close freind's of mine family. He let me know that him and his family were looking to engage with their customers more effectivly through thier online platform, but felt that thier current website was outdated and that they were looking for a redesign. Because of this, a group of designers including myself decided to redesign the three different portions of the current website to give them options to pick thier favorite design.

My Role:

I took responsiblity of redesigning their current the contact page in order to make it easier for the user to engage with the page when it comes to content and accesibility and provide options for them to contact the resturant. To do this, I first identified usability issues, created a visual style guide, then encoded a redesigned webpage would look like, which can be found here.

The Process and Results:

This redesign process was broken into two parts:

1: Identify usability when it came to efficiency, learnability, memorability, and conceptual model of the page

2.1: Create a visual style guide and mockups of the redesigned page to address these usability issues

2.2: Use html and css to code what this redesigned webpage would look like following this style guide

Part 1: Identifying Usability Issues

In order to understand how to create a valid color palette redesign for the this webpage, I had to first understand the current issues when it came to the webpage's usability. A summary of some of the major issues I found are listed below.

  • The page does not contain screen reader capabilities making it inaccessible
  • The image on this page that has little in relation to the contact information of the restaurant, making the page slightly inefficent
  • Font text of the same information varied across the page
  • No alternative text or labels were integrated into the webpage

Addressing Issues:

  • Added a photo of the owner which relevant to the resturant and contact page
  • Utilized a consistent font throughout the page
  • Used headings throughout the page in order to create visual heiarchy
  • Encoded keyboard naviagtion and speech reader capabilites (within website)
  • Incldued a section where the user could input an message and send an request or communicate with the restaurant

Responseiveness:

In these mockups I adjusted the platforms for things to appear differently based on the different-sized screens and modes of access used by the user.

  • Larger Screens -> Horizontal layout in order to minimize scroll and fill up the space of the screen with a clear region of seperation between static information and the messaging portion
  • Smaller Screens -> Vertical layout as well smaller text in order to fit things on smaller screens while still allowing the user to effectivly engage with the information
